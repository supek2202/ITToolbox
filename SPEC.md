# 网络设备巡检工具 - 规格说明书

## 1. 项目概述

**项目名称**: NetInspector - 网络设备巡检工具  
**项目类型**: 桌面应用程序  
**核心功能**: 支持路由器、交换机、防火墙等网络设备的自动发现、连接、巡检命令执行和报告生成  
**目标用户**: 网络工程师、系统管理员

## 2. UI/UX 规格

### 2.1 窗口结构

- **主窗口**: 1200x800 像素，可调整大小
- **布局**: 左侧设备列表 + 右侧详情/操作面板 + 底部日志输出

### 2.2 视觉设计

- **主色调**: #1E3A5F (深蓝)
- **次要色**: #2D5A87 (中蓝)
- **强调色**: #4CAF50 (绿色 - 表示在线/成功)
- **警告色**: #FF9800 (橙色)
- **危险色**: #F44336 (红色 - 表示离线/错误)
- **背景色**: #F5F5F5 (浅灰)
- **文字色**: #333333 (深灰)
- **字体**: Arial, 12px 正文, 14px 标题

### 2.3 组件列表

1. **顶部工具栏**
   - 发现设备按钮
   - 执行巡检按钮
   - 导出报告按钮
   - 设置按钮

2. **左侧设备表格**
   - 列: 序号、设备类型、厂商、型号、IP、状态、最后巡检时间
   - 支持排序、筛选
   - 右键菜单: 编辑、删除、执行巡检

3. **右侧设备详情面板**
   - 设备信息表单
   - 巡检命令选择
   - 巡检结果查看

4. **底部日志区域**
   - 显示操作日志
   - 支持滚动查看

## 3. 功能规格

### 3.1 设备管理

- **手动添加设备**: 填写设备类型、厂商、型号、IP、端口、用户名、密码、enable密码
- **自动发现设备**: 扫描IP段发现网络设备（通过ping和常见端口22/23/443）
- **设备类型**: 路由器、交换机、防火墙、无线控制器、负载均衡器、其他
- **厂商支持**: Cisco、Huawei、H3C、Juniper、Fortinet、锐捷、深信服、其他
- **设备表格数据持久化**: JSON文件存储

### 3.2 设备发现

- **IP段扫描**: 支持输入如 192.168.1.1-254 或 192.168.1.0/24
- **端口扫描**: 检测22(SSH)、23(Telnet)、443(HTTPS)端口
- **厂商识别**: 根据banner信息识别厂商
- **并行扫描**: 多线程加速扫描

### 3.3 巡检执行

- **支持协议**: SSH (主要)、Telnet
- **巡检命令库**: 可配置的JSON文件
- **默认巡检命令**:
  - Cisco: show version, show interface, show ip route, show cpu, show memory等
  - Huawei: display version, display interface, display ip routing-table等
  - 防火墙: 查看会话表、CPU、内存、连接数等
- **并行执行**: 多设备同时巡检
- **结果保存**: 保存到JSON文件和HTML报告

### 3.4 巡检命令管理

- **命令库文件**: commands.json
- **可配置项**: 厂商、设备类型、命令描述、命令内容
- **支持变量**: {hostname} 等
- **新增/修改**: 通过UI或直接编辑JSON

### 3.5 报告生成

- **格式**: HTML网页报告
- **内容**: 设备信息、巡检命令、输出结果、统计信息
- **导出**: 保存到指定目录

## 4. 技术实现

### 4.1 技术栈

- **GUI**: tkinter (Python内置)
- **网络连接**: netmiko
- **并发**: concurrent.futures
- **数据存储**: JSON
- **打包**: PyInstaller

### 4.2 打包为可执行文件

- 使用 PyInstaller 生成 Windows/Mac 可执行文件
- 包含所有依赖，无需目标机器安装Python

## 5. 验收标准

1. ✅ 可添加、编辑、删除设备
2. ✅ 可扫描IP段发现设备
3. ✅ 可连接设备并执行巡检命令
4. ✅ 巡检结果保存到文件
5. ✅ 可新增/修改巡检命令
6. ✅ 可生成HTML报告
7. ✅ 打包成可执行文件，在无Python环境下运行
